DESIGN SPEC — Unified Pill System

Problem: Row-3 had inconsistent element sizes — badges (22px), buttons (28px), XP bar (4px), PvP text (16px). Each state created different height and visual rhythm = chaos.


Solution: Unified Pill System. Every element in row-3 is a .pill with identical structure:

Pill Token Spec (8px grid)
height: 28px (3.5 × 8)
border-radius: 8px (1 × 8)
padding: 0 10px
font-size: 11px (LayoutConstants.textBadge)
font-weight: 600
gap: 4px (internal elements)
icon: 12px
border: 1px solid (color-dependent)
Pill Types
🧪Heal×3
🧪Heal Now!×3
⚡Restore×2
⭐+3 Points
⚠Broken
💀Critical
🔥5
⭐1st Win
🏆1633
Interactive pills (.pill-act) have cursor:pointer + hover brightness.
Info pills are non-interactive. Same height, same visual weight.
Widget Structure (Fixed)
Row 1: Name (Oswald 16px) + Class (Inter 11px) — always same
Row 2: HP bar (8px) + HP text (10px) — always same
Row 3: Pills or XP bar — always 28px height

Row 3 content is contextual but height is constant.
No layout shift between states. No visual chaos.
Mobile UX Compliance
✅ Avatar: 48×48px (≥ LayoutConstants.touchMin)
✅ Refill button: 24×24px visual, 48px tap area via padding
✅ Action pills: 28px height, adequate tap area
✅ All text ≥ 10px (≥ LayoutConstants.textBadge - 1)
✅ HP bar: 8px height (visible, tappable area via row)
✅ 8px grid alignment throughout
✅ Row-3 flex-wrap: safe on narrow screens
✅ Widget min-height: 76px — no collapse
✅ Single-column widget, works 320px-428px
✅ No hover-only interactions
Row-3 Priority Logic
1. Critical HP + potion → pill-heal-urgent
2. Critical HP no potion → pill-warn "Critical"
3. Low HP + potion → pill-heal
4. Low stamina + potion → pill-energy
5. Stat points > 0 → pill-stat
6. Broken gear → pill-warn "Broken"
7. Arena context → pill-pvp + pill-streak + pill-bonus
8. Default → XP bar

Max 3 pills visible. Priority 1-6 override XP bar.
Multiple warnings can coexist (e.g. stat + broken).
