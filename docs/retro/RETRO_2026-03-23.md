## Паттерны и уроки

### 1. `.foregroundColor()` sweep неполный
**Что:** Коммит `38bd758` мигрировал `InboxRowView` (12 вызовов), но пропустил `InboxDetailView` (7 вызовов) в той же папке.
**Причина:** Ручной sweep без полного файлового сканирования.
**Правило:** При выполнении массовой миграции (`.foregroundColor()` → `.foregroundStyle()`) — всегда сканировать ВСЁ дерево и фиксить все файлы за один коммит.
**Агенты:** Guardian scanner уже ловит `.foregroundColor()` — правила менять не нужно. Нужен ручной фикс `InboxDetailView.swift`.

### 2. `Color(hex:)` в LevelUpModalView
**Что:** `LevelUpModalView.swift:200` всё ещё использует `Color(hex: 0xB8860B)` вместо `DarkFantasyTheme.goldDim`.
**Причина:** Файл сильно переработан в `cd2a886`, но одно старое значение осталось.
**Действие:** Нужен точечный фикс — заменить на токен.

### 3. OrnamentalTitle — паттерн explicit init
**Что:** Новый компонент `OrnamentalTitle` потребовал отдельного коммита для добавления `init(_ title:)` потому что Swift memberwise init не поддерживает unlabeled arguments.
**Урок:** При создании новых SwiftUI view-struct'ов с одним primary parameter — ВСЕГДА добавлять explicit `init(_ paramName:)` сразу, не полагаясь на memberwise init.
**Статус:** Уже исправлено в коммите `a4663b1`.

### 4. Prisma Json double-cast — рекуррентный паттерн
**Что:** Коммит `952570f` — очередной случай забытого double-cast для Prisma `Json` fields.
**Статус:** Правило уже в CLAUDE.md (`## Backend TypeScript Rules`). Работает — ошибка была поймана и пофикшена быстро.
