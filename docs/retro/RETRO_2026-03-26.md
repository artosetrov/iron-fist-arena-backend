## Паттерны и уроки

### 1. Asset Catalog `provides-namespace` — новый паттерн

**Что случилось:** Подпапки `Items/` и `Skins/` в Assets.xcassets имели `"provides-namespace": true` в `Contents.json`. Это значит, что `UIImage(named: "potion-hp")` не работает — нужно `UIImage(named: "Items/potion-hp")`. Код никогда не использовал prefix, поэтому все картинки не отображались.

**Урок:** При создании подпапок в Assets.xcassets **не включать** `provides-namespace`, если код обращается к ассетам по короткому имени без prefix.

**Действие:** Добавить правило в CLAUDE.md.

### 2. Timer.publish → MainActor Task (Swift 6 Sendable)

**Что случилось:** `Timer.publish(every:on:in:)` в комбинации с `.onReceive` порождает Sendable/nonisolated warnings в Swift 6 strict concurrency, т.к. Timer callback не гарантирует MainActor isolation.

**Текущее состояние:** 2 оставшихся использования Timer.publish:
- `GoldMineDetailView.swift:274` — onReceive(Timer.publish(every: 30...))
- `EventBannerView.swift:28` — Timer.publish(every: 1...)

**Урок:** Для периодических обновлений UI предпочитать `Task { while !Task.isCancelled { try await Task.sleep(...) } }` с `@MainActor` context.

### 3. Apple Sign In — 3 итерации до рабочего варианта

**Что случилось:** Invisible overlay (opacity 0.01) для SignInWithAppleButton не работал надёжно — blendMode ломал tap detection, низкая opacity не регистрировала нажатия. Переписали на программный `ASAuthorizationController`.

**Урок:** `SignInWithAppleButton` ненадёжен как invisible overlay. Программный `ASAuthorizationController` — единственный стабильный путь для кастомных кнопок Apple Sign In.

### 4. SFX.uiSuccess не существует

**Что случилось:** `LevelUpModalView` использовал `SFX.uiSuccess`, который не существует в enum. Правило в CLAUDE.md перечисляет `.uiTap`, `.uiConfirm`, `.uiTapHeavy` и т.д., но не содержит **полного списка** доступных кейсов.

**Действие:** Обновить CLAUDE.md — добавить полный список SFX enum cases.

### 5. [weak self] в struct — рецидив

Коммит `4821ec0` исправил `[weak self]` в `HeroDetailView` (struct). Правило уже в CLAUDE.md, но нарушение всё ещё случается. Текущий grep в guardian scanner уже проверяет это — возможно, был добавлен в сессии без запуска guardian.
