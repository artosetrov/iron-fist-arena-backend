## Уроки

### 1. TOCTOU в PvP — stamina check вне транзакции
- **Что:** pvp/fight и pvp/resolve проверяли stamina ВНЕ транзакции. Два параллельных запроса = бесплатные бои.
- **Причина:** Изначальный код следовал "optimistic check → transaction write" без row lock.
- **Правило:** ВСЕ проверки ресурсов (stamina, gold, gems) ОБЯЗАНЫ быть ВНУТРИ `$transaction` с `FOR UPDATE`.
- **Обновлено:** CLAUDE.md (уже было правило для shop, теперь расширено на PvP).

### 2. Invented tokens в новых файлах
- **Что:** SessionSummaryView и RankUpCeremonyView создавались с `.bodyFont`, `.spacingLG` — несуществующими токенами.
- **Причина:** Агент создающий файл не сверился с DarkFantasyTheme.swift.
- **Правило:** УЖЕ в CLAUDE.md. Нужно усилить: CDO Phase 6 обязателен для каждого нового .swift файла.

### 3. Public API routes без rate limiting
- **Что:** leaderboard, search, profile — открыты для scraping.
- **Причина:** Изначально auth не требовался (публичные данные), rate limit забыли.
- **Правило:** Все public GET routes ОБЯЗАНЫ иметь IP-based rate limiting.

### 4. Character delete без cascade cleanup
- **Что:** `prisma.character.delete()` падал с FK violation на PvpMatch.
- **Причина:** PvpMatch → Character FK не имеет `onDelete: Cascade` (и не должен — matches это история).
- **Правило:** Перед delete cascade-free моделей — manual cleanup related records.

### 5. Git rebase с untracked файлами
- **Что:** `git rebase --abort` не работал из-за untracked файлов конфликтующих с целевым коммитом.
- **Причина:** Skills и assets были untracked на одной ветке, tracked на другой.
- **Решение:** Удалить .git/rebase-merge/ + lock файлы вручную, затем checkout target.

### 6. Shell Game без серверного лимита
- **Что:** Бесконечный grind через shell game (RTP=50%, нет daily cap).
- **Причина:** Лимит не был в спеке.
- **Правило:** Все minigames ОБЯЗАНЫ иметь серверный daily limit.
