Integrated Hero Card Structure
Header (~100pt):
  Identity: Name · Class · Level
  HP bar: full-width, 10pt, 12px font
  XP bar: full-width, 6pt, secondary
  Resources: ⚡ stamina | 🪙 gold | 💎 gems
  Actions: [Stance pill] [Repair pill] [Stat pill] [Heal pill]

Divider (1px)

Equipment Grid (~220pt):
  3 left armor slots (64×64) | Portrait (flex, ~208pt) | 3 right armor slots
  Bottom row: 5 compact slots (56×56) — Weapon, Relic, Amulet, Ring, Ring

Total card height: ~330pt vs previous 550pt (widget + equip) = −40%
What Was Eliminated
✅ UnifiedHeroWidget removed from Hero page (stays on Hub/Arena/Dungeon)
✅ Stance card 56pt → inline pill 28pt in action row
✅ HP/XP bars: no longer duplicated (shown once in header)
✅ Name/Level/Class: shown once in header, not on portrait
✅ Empty bottom row cells: eliminated (5 slots in 1 row)
✅ Portrait clean: no name overlay, no info clutter
Action Pill Priority (in actions row)
1. 🧪 Heal pill — if HP < 50% and has health potion
2. ⚠ Repair pill — if broken gear exists (count + total cost)
3. ⭐ Stat pill — if statPoints > 0
4. ⚔🛡 Stance pill — always (compact, inline)

Max visible: 3 pills + stance. If no actions needed, stance only.
Mobile UX Compliance
✅ Armor slots: 64×64 ≥ 44pt touch min
✅ Bottom slots: 56×56 ≥ 44pt touch min
✅ All text ≥ 12px (WCAG minimum)
✅ HP bar: full-width, readable
✅ Resources: single line, scannable
✅ Action pills: 28px height, flex-wrap safe
✅ 4px grid compliance throughout
✅ Portrait dominant: fills remaining width (~55%)
